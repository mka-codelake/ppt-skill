Storyline Patterns
==================

Starting skeletons for common presentation types. They are scaffolds, not
templates -- always adapt to the concrete content, audience and decision.
Each maps onto the SCR narrative (see methodology.md) and ends with one
explicit call to action.

Pitch / proposal
----------------

```
Title
Agenda
-----------------
Chapter: Starting point
  -> Executive summary (SCR on one slide)
  -> The client's challenge (deepen the complication)
-----------------
Chapter: Our approach
  -> Solution overview (unfold the resolution)
  -> Method / phases
  -> Why us / differentiator
-----------------
Chapter: Value
  -> Concrete benefit for the client
  -> References / proof
-----------------
Chapter: Next steps
  -> Approach / timeline
  -> Investment
-----------------
Contacts
Closing (call to action)
```

Strategy / decision proposal
----------------------------

```
Title
Agenda
-----------------
Chapter: Situation & target picture
  -> Current situation (Situation)
  -> Pressure to act (Complication)
  -> Strategic direction (Resolution / core message)
-----------------
Chapter: Analysis
  -> Market / competition / internal view
  -> Key findings (3 main arguments)
-----------------
Chapter: Options
  -> Option A vs. Option B (two-column)
  -> Assessment & recommendation
-----------------
Chapter: Execution
  -> Roadmap / phases
  -> Resources & risks
-----------------
Chapter: Decision
  -> Recommendation summary
  -> Call to action / next steps
-----------------
Closing
```

Concept / workshop
------------------

```
Title
Agenda
-----------------
Chapter: Context & goals
  -> Why are we here? (Situation + Complication)
  -> Goals of today's session
-----------------
Chapter: Concept
  -> Core idea (core message)
  -> Building blocks (one slide per element)
-----------------
Chapter: Discussion & validation
  -> Open questions
  -> Decisions needed
-----------------
Chapter: Next steps
  -> Concrete actions with owners and dates
-----------------
Closing
```

Ghost deck
----------

Whatever the pattern, the slide phases (4-6) produce a **ghost deck**: one
slide per message, action title set, body empty. Approve the ghost deck
(title-reading test) before any exhibit is built -- that is the point at
which the story becomes slides, and it is the artefact handed to the `ppt`
skill to build.
