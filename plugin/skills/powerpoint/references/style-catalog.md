Style Catalog
=============

Two styles are chosen ONCE per deck (before any image prompt is
written) and then applied verbatim to every prompt, so all visuals
look like one family. Curated options below; free-text descriptions
are equally valid -- if the user describes a style, use it verbatim.

Image Style (photos / motifs)
-----------------------------

| Style          | Prompt block                                                              |
|----------------|---------------------------------------------------------------------------|
| Stock Photo    | `commercial photography, shot on Canon EOS R5 85mm f/1.4, soft diffused light, shallow depth of field, professional color grading, 8k` |
| Editorial      | `editorial photography, shot on Hasselblad 120mm, cinematic color grading, golden hour backlighting, 8k` |
| Illustration   | `modern flat vector illustration, clean shapes, subtle gradients, generous negative space` |
| 3D Render      | `polished 3D render, soft studio lighting, matte materials, octane render quality` |

Info-Graphic Style (diagrams / data visuals)
--------------------------------------------

| Style       | Prompt block                                                            |
|-------------|--------------------------------------------------------------------------|
| Flat        | `flat design infographic, solid color fills, clean sans-serif labels, no shadows` |
| Line        | `minimal line-art infographic, thin consistent strokes, outline icons`   |
| Isometric   | `isometric infographic, 30-degree perspective, consistent depth, soft shadows` |
| Hand-drawn  | `hand-drawn sketch infographic, marker strokes, whiteboard aesthetic`    |

Consistency rules
-----------------

-   The chosen image-style block appears verbatim in EVERY photo/motif
    prompt; the chosen info-graphic block in EVERY diagram prompt.
-   Never mix styles within one deck unless the user explicitly asks.
-   Template sidecar constraints (e.g. dark backgrounds on title and
    closing slides for illustration styles) take precedence over the
    style block where they conflict.
